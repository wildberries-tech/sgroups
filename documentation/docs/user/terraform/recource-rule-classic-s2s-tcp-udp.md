## **Описание**

sgroups_rules - это ресурс Terraform для определения общих правил безопасности между двумя группами безопасности.

## **Параметры Ресурса**

| название | обязательность | тип данных | доп. описание |
| --- | --- | --- | --- |
| transport | да | "TCP\|UDP" | Тип протокола для данного правила. Должен быть установлен в "TCP" или "UDP". |
| logs | нет | bool | Определяет, включены ли логи для данного правила. По умолчанию значение "false". Установите в "true", если требуется включить логирование. |
| trace | нет | bool | Определяет, включен ли трассировочный режим для данного правила. По умолчанию значение "false". Установите в "true", если требуется включить трассировку. (пока не реализовано) |
| sg_from | да | string | Имя исходной группы безопасности. |
| sg_to | да | string | Имя целевой группы безопасности. |
| ports[] | нет | array of object | Массив объектов, представляющих порты. Каждый объект содержит поля s (source) и d (destination). |
| ports[].s | нет | string | Значения s (source) портов в рамках одного правила не должны пересекаться |
| ports[].d | нет | string | Значения d (destination) портов в рамках одного правила могут пересекаться |

## **Пример использования**

```hcl
resource "sgroups_rules" "rules" {
    items = {
      "TCP:sg(sg-1)sg(sg-2)" = {
        transport = "TCP"
        logs = true
        trace = true
        sg_from = "sg-1"
        sg_to = "sg-2"
        ports = [
          { s = "80", d = "8080" }
          ]
      }
    }
}
```

## **Ограничения**

* Каждое правило должно иметь уникальный ключ, чтобы избежать пересечений.
* Структура формы items должна соответствовать определенной последовательности которой необходимо придерживаться `"${transport}:sg(${sgroup_from})sg(${sgroup_to})"`
* sg_from и sg_to - длина значения не должна превышать 256 символов. Значение должно начинаться и заканчиваться символами без пробелов.
* Значения s (source) портов не должно пересекаться в рамках одного правила состоящего из уникального составного ключа (transport, sg_from, sg_to).
* Значения s (source) и d (distinct) портов должно находиться в интервале от 1 до 65535. Если значение не будет указано то будет использоваться весь диапазон портов.
