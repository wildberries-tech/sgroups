## **Описание**

sgroups_icmp_rules - это ресурс Terraform для определения правил безопасности ICMP между двумя группами безопасности.

## **Параметры Ресурса**

| название | обязательность | тип данных | доп. описание |
| --- | --- | --- | --- |
| logs | нет | bool | Определяет, включены ли логи для данного правила. По умолчанию значение "false". Установите в "true", если требуется включить логирование. |
| trace | нет | bool | Определяет, включен ли трассировочный режим для данного правила. По умолчанию значение "false". Установите в "true", если требуется включить трассировку. |
| sg_from | да | string | Имя исходной группы безопасности. |
| sg_to | да | string | Имя целевой группы безопасности. |
| ip_v | да | "IPv4\|IPv6" | Версия IP для ICMP (IPv4 или IPv6). |
| type | да | array of strings | Список, определяющий допустимые типы ICMP запросов. |

## **Пример использования**

```hcl
resource "sgroups_icmp_rules" "rules" {
    items = {
      "sg(sg-1)sg(sg-2)icmpIPv4" = {
        logs = true
        trace = true
        sg_from = "sg-1"
        sg_to = "sg-2"
        ip_v = "IPv4"
        type = [
         0, 8
         ]
        }
    }
}
```

## **Ограничения**

* Каждое правило должно иметь уникальный ключ, чтобы избежать пересечений.
* Структура формы items должна соответствовать определенной последовательности которой необходимо придерживаться `"sg(${sgroup_from})sg(${sgroup_to})icmp${split("icmpIPv", protocol).1}"`
* sg_from и sg_to - длина значения не должна превышать 256 символов. Значение должно начинаться и заканчиваться символами без пробелов.
* ip_v - может быть только одно из двух значений "IPv4" или "IPv6".
* type - значения в списке не должны повторяться. Значение должно быть цифрой от 0 до 255.
