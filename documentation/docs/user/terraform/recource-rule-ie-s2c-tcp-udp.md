## **Описание**

sgroups_cidr_rules - это ресурс Terraform для определения правил безопасности на основе CIDR блоков, групп безопасности, протокола и портов.

## **Параметры Ресурса**

| название | обязательность | тип данных | доп. описание |
| --- | --- | --- | --- |
| traffic | да | "ingress\|egress" | Направление трафика для данного правила. ingress - входящий траффик. egress - исходящий траффик. |
| sg_name | да | string | Имя группы безопасности. |
| cidr | да | string | CIDR блок для данного правила. |
| transport | да | "TCP\|UDP" | Тип протокола для данного правила. Должен быть установлен в "TCP" или "UDP". |
| logs | нет | bool | Определяет, включены ли логи для данного правила. По умолчанию значение "false". Установите в "true", если требуется включить логирование. |
| trace | нет | bool | Определяет, включен ли трассировочный режим для данного правила. По умолчанию значение "false". Установите в "true", если требуется включить трассировку. (пока не реализованно) |
| ports[] | нет | array of object | Массив объектов, представляющих порты. Каждый объект содержит поля s (source) и d (destination). |
| ports[].s | нет | string | Значения s (source) портов в рамках одного правила не должны пересекаться |
| ports[].d | нет | string | Значения d (destination) портов в рамках одного правила могут пересекаться |

## **Пример использования**

```hcl
resource "sgroups_cidr_rules" "rules" {
    items = {
      "TCP:cidr(192.168.1.0/24)sg(sg-1)ingress" = {
        traffic = "ingress"
        sg_name = "sg-1"
        cidr = "192.168.1.0/24"
        transport = "TCP"
        logs = true
        trace = false
        ports = [
          { s = "80", d = "8080" }
          ]
        }
    }
}
```

## **Ограничения**

* Каждое правило должно иметь уникальный ключ, чтобы избежать пересечений.
* Структура формы items должна соответствовать определенной последовательности которой необходимо придерживаться `"${transport}:cidr(${cidr})sg(${sg_name})${traffic}"`
* sg_name - длина значения не должна превышать 256 символов. Значение должно начинаться и заканчиваться символами без пробелов.
* Значения s (source) портов не должно пересекаться в рамках одного правила состоящего из уникального составного ключа (transport, sg_name, traffic).
* Значения s (source) и d (distinct) портов должно находиться в интервале от 1 до 65535. Если значение не будет указано то будет использоваться весь диапазон портов.
* Значение CIDR не должно пересекаться в рамках одного правила состоящего из уникального составного ключа (transport, sg_name, traffic).
