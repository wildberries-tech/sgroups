# Конфигурация  правил

## Множества (Sets)

#### Описание

<div class="paragraph">
Множества - это контейнеры элементов определяемого пользователем типа данных, они однозначно идентифицируются определяемым пользователем именем и прикрепляются к таблицам. В нашем случае это может быть либо FQDN, либо другая Secutrity Group либо CIDR.
</div>

#### Параметры

<table>
    <thead>
        <tr>
            <th>Наименование параметра</th>
            <th>Описание</th>
            <th>Значения</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>family</td>
            <td>Семейства адресов определяют тип обрабатываемых пакетов. Для каждого семейства адресов ядро ​​содержит так называемые перехватчики на определенных этапах пути обработки пакетов, которые вызывают nftables, если существуют правила для этих перехватчиков.</td>
            <td>Значение типа string, не должно содержать в себе пробелов</td>
        </tr>
        <tr>
            <td>table</td>
            <td>Наименование множества, будь то Security Group или FQDN</td>
            <td>Значение типа string, не должно содержать в себе пробелов</td>
        </tr>
        <tr>
            <td>table.type</td>
            <td>Тип данных элементов множества.</td>
            <td>Могут быть установлены следующие значения: ipv4_addr, ipv6_addr, ether_addr, inet_proto, inet_service, mark</td>
        </tr>
        <tr>
            <td>table.flags</td>
            <td>Значения используются для описания свойств множеств.</td>
            <td>Могут быть установлены следующие значения: constant, dynamic, interval, timeout.</td>
        </tr>
        <tr>
            <td>table.elements</td>
            <td>Элементы, содержащиеся в наборе в формате CIDR</td>
            <td>Значения CIDR, в случае нескольких значений перечисляются через запятую</td>
        </tr>
    </tbody>
</table>

#### Пример

```hcl
set familyName_example-sgName_example {
		type ipv4_addr
		flags constant,interval
		elements = { 10.168.24.0/23 }
	}
```

#### Приоритеты обработки цепочек

<div class="paragraph">
В цепочке можно указать хук и значение приоритета. Параметр приоритета задаётся названием приоритета или его численным значением. Цепочки с более низкими значениями обрабатываются раньше. Значение приоритета может быть отрицательным.
</div>

#### Пример

```hcl
chain INPUT {
		type filter hook input priority 0;
	}
```

## Verdict statement {#verdict_statment}

<table>
    <thead>
        <tr>
            <th>Значение</th>
            <th>Описание</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>accept</td>
            <td>Завершает оценку набора правил и принимает пакет. Пакет по-прежнему может быть отброшен позже с помощью другого перехватчика, например, принятие в прямом перехватчике по-прежнему позволяет отбросить пакет позже в перехватчике постмаршрутизации или другой прямой базовой цепочке, которая имеет более высокий номер приоритета и впоследствии оценивается в конвейере обработки.</td>
        </tr>
        <tr>
            <td>drop</td>
            <td>Завершает оценку набора правил и отбрасывает пакет. Отбрасывание происходит мгновенно, никакие дальнейшие цепочки или хуки не оцениваются. Невозможно снова принять пакет в более поздней цепочке, поскольку они больше не оцениваются в рамках пакета.</td>
        </tr>
        <tr>
            <td>jump *chain*</td>
            <td>Продолжает оценку с первого правила в цепочке. Текущая позиция в наборе правил помещается в стек вызовов, и оценка продолжится там, когда новая цепочка будет полностью оценена или будет выдан обратный вердикт. В случае, если правило в цепочке выносит абсолютный вердикт, оценка набора правил немедленно прекращается и предпринимаются определенные действия.</td>
        </tr>
        <tr>
            <td>goto *chain*</td>
            <td>Аналогично jump, но текущая позиция не помещается в стек вызовов, а это означает, что после новой цепочки вычисление продолжится с последней цепочки, а не с той, которая содержит оператор goto.</td>
        </tr>
    </tbody>
</table>
