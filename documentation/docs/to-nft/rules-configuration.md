# Конфигурация  правил

## Множества (Sets)

#### Описание

<div class="paragraph">
Множества - это контейнеры элементов определяемого пользователем типа данных, они однозначно идентифицируются определяемым пользователем именем и прикрепляются к таблицам. В нашем случае это может быть либо FQDN, либо другая Secutrity Group либо CIDR.
</div>

#### Параметры

| Наименование параметра | Описание | Значения |
| --- | --- | --- |
| family | Семейства адресов определяют тип обрабатываемых пакетов. Для каждого семейства адресов ядро ​​содержит так называемые перехватчики на определенных этапах пути обработки пакетов, которые вызывают nftables, если существуют правила для этих перехватчиков. | Значение типа string, не должно содержать в себе пробелов |
| table | Наименование множества, будь то Security Group или FQDN | Значение типа string, не должно содержать в себе пробелов |
| table.type | Тип данных элементов множества. | Могут быть установлены следующие значения: ipv4_addr, ipv6_addr, ether_addr, inet_proto, inet_service, mark |
| table.flags | Значения используются для описания свойств множеств. | Могут быть установлены следующие значения: constant, dynamic, interval, timeout. |
| table.elements | Элементы, содержащиеся в наборе в формате CIDR | Значения CIDR, в случае нескольких значений перечисляются через запятую |

#### Пример

```hcl
set familyName_example-sgName_example {
		type ipv4_addr
		flags constant,interval
		elements = { 10.168.24.0/23 }
	}
```



## Цепочки (Chains)

| Наименование цепочки | Описание |
| --- | --- |
| FW-IN | Основная цепочка для правил по умолчанию для входящего трафика |
| FW-OUT | Основная цепочка для правил по умолчанию для исходящего трафика |
| POSTROUTING | Цепочка для постобработки исходящего трафика |
| INPUT | Цепочка для постобработки входящего трафика |
| FW-IN-(Security Group name) | Цепочка для входящего трафика группы безопасности |
| FW-OUT-(Security Group name) | Цепочка для исходящего трафика группы безопасности |

#### Приоритеты обработки цепочек

<div class="paragraph">
В цепочке можно указать хук и значение приоритета. Параметр приоритета задаётся названием приоритета или его численным значением. Цепочки с более низкими значениями обрабатываются раньше. Значение приоритета может быть отрицательным.
</div>

#### Пример

```hcl
chain INPUT {
		type filter hook input priority 0;
	}
```

## Правила (Rules)

### ICMP Global

#### Параметры

| Наименование параметра | Описание | Значения |
| --- | --- | --- |
| nftrace | Трассировка указанного правила (опциональна) | В случае если трассировка включена "nftrace set 1" |
| icmp type |  |  |
| counter packets *number* bytes *number* | Счетчик, учитывает количество пройденных пакетов с количеством байтов переданной информации в рамках указанной цепочки правил |  |
| verdict_statement | Вердикт политики по пакетам данных | Может принимать одно из следующих значений:<br />\{accept | drop\} \{jump | goto\} *chain*<br />более подробное описание в таблице [Verdict statement](https://youtrack.wildberries.ru/articles/SWARM-A-149/Konfiguraciya-pravil#verdict-statement) |

#### Пример использования

```
nftrace set 1 icmp type { echo-reply, echo-request } counter packets 0 bytes 0 log level debug flags ip options accept
```

## Verdict statement

| Значение | Описание |
| --- | --- |
| accept | Завершает оценку набора правил и принимает пакет. Пакет по-прежнему может быть отброшен позже с помощью другого перехватчика, например, принятие в прямом перехватчике по-прежнему позволяет отбросить пакет позже в перехватчике постмаршрутизации или другой прямой базовой цепочке, которая имеет более высокий номер приоритета и впоследствии оценивается в конвейере обработки. |
| drop | Завершает оценку набора правил и отбрасывает пакет. Отбрасывание происходит мгновенно, никакие дальнейшие цепочки или хуки не оцениваются. Невозможно снова принять пакет в более поздней цепочке, поскольку они больше не оцениваются в рамках пакета. |
| jump *chain* | Продолжает оценку с первого правила в цепочке. Текущая позиция в наборе правил помещается в стек вызовов, и оценка продолжится там, когда новая цепочка будет полностью оценена или будет выдан обратный вердикт. В случае, если правило в цепочке выносит абсолютный вердикт, оценка набора правил немедленно прекращается и предпринимаются определенные действия. |
| goto *chain* | Аналогично jump, но текущая позиция не помещается в стек вызовов, а это означает, что после новой цепочки вычисление продолжится с последней цепочки, а не с той, которая содержит оператор goto. |
