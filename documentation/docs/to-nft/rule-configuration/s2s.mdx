                #### Пример использования

                ```yaml
                name: sg-example-from
                rules:
                    s2s:
                    - sgroupSet:
                        - sg-example-to
                        logs:  true
                        trace: true
                        access:
                            tcp:
                            - description: ""
                            ports_from:
                                - 64231
                            ports_to:
                                - 443
                                - 80
                            udp:
                            - description: ""
                            ports_from:
                                - 64231
                            ports_to:
                                - 443
                                - 80        
                ```

                ## Ограничения

                <ul>
                    <li>Необходимо указать минимум одно значение в sgroupSet.</li>
                    <li class="text-justify">sgroupSet - длина одного значения из списка не должна превышать 256 символов. Значение должно начинаться и заканчиваться символами без пробелов.</li>
                    <li>Может быть использован только один протокол для текущего правила (TCP или UDP).</li>
                    <li class="text-justify">Значения `rules.s2s[].access.tcp/udp.ports_to[]` и `rules.s2s[].access.tcp/udp.ports_from[]` портов должно находиться в интервале от 1 до 65535. Если значение не будет указано то будет использоваться весь диапазон портов.</li>
                    <li>Значения `rules.s2s[].access.tcp/udp.ports_from[]` портов не должно пересекаться в рамках текущего правила.</li>
                </ul>
            </TabItem>

                #### Пример использования

                ```hcl
                resource "sgroups_ie_rules" "rules" { 
                    items = { 
                        "tcp:sg(sg-example-from)sg(sg-example-to)" = {
                            transport   = "tcp"
                            sg_from     = "sg-example-from"
                            sg_to       = "sg-example-to"
                            ports = [
                                {
                                    d = "443,80"
                                    s = "64231"
                                }
                            ]
                            logs  = true
                            trace = true
                        }
                        "udp:sg(sg-example-from)sg(sg-example-to)" = {
                            transport   = "udp"
                            sg_from     = "sg-example-from"
                            sg_to       = "sg-example-to"
                            ports = [
                                {
                                    d = "443,80"
                                    s = "64231"
                                }
                            ]
                            logs  = true
                            trace = true
                        }                
                    }
                }        
                ```

                #### Ограничения

                <ul>
                    <li>Каждое правило должно иметь уникальный ключ, чтобы избежать пересечений.</li>
                    <li>Структура формы items должна соответствовать определенной последовательности которой необходимо придерживаться `"${transport}:sg(${sgroup_from})sg(${sgroup_to})"`</li>
                    <li>sg_from и sg_to - длина значения не должна превышать 256 символов. Значение должно начинаться и заканчиваться символами без пробелов.</li>
                    <li>Значения s (source) портов не должно пересекаться в рамках одного правила состоящего из уникального составного ключа (transport, sg_from, sg_to).</li>
                    <li>Значения s (source) и d (distinct) портов должно находиться в интервале от 1 до 65535. Если значение не будет указано то будет использоваться весь диапазон портов.</li>
                </ul>

                ```hcl
                chain FW-IN-sgName {
                    # **********
                    ${Trace} ${RuleType} ${SrcSgroup} ${Transport} ${SrcPorts} ${DstPorts} ${Counter} ${Log} ${Verdict}
                    # **********
                }
                ```

                ```hcl
                chain FW-OUT-sgName {
                    # **********
                    ${Trace} ${RuleType} ${DstSgroup} ${Transport} ${SrcPorts} ${DstPorts} ${Counter} ${Log} ${Verdict}
                    # **********
                }
                ```

                ```hcl
                chain FW-IN-sg-example-to {
                    # **********
                ```          

               #### Пример использования

                ```hcl
                resource "sgroups_icmp_rules" "rules" {
                    items = {
                        "sg(sg-example-from)sg(sg-example-to)icmp4" => {
                            logs    = true
                            trace   = true
                            sg_from = sg-example-from
                            sg_to   = sg-example-to
                            ip_v    = IPv4
                            type    = [0,8]
                        }
                        "sg(sg-example-from)sg(sg-example-to)icmp6" => {
                            logs    = true
                            trace   = true
                            sg_from = sg-example-from
                            sg_to   = sg-example-to
                            ip_v    = IPv6
                            type    = [0,8]
                        }
                    }
                }
                ```  

                #### Ограничения

                <ul>
                    <li>Каждое правило должно иметь уникальный ключ, чтобы избежать пересечений.</li>
                    <li>Структура формы items должна соответствовать определенной последовательности которой необходимо придерживаться `"sg(${sgroup_from})sg(${sgroup_to})icmp${split("icmpIPv", protocol).1}"`</li>
                    <li>sg_from и sg_to - длина значения не должна превышать 256 символов. Значение должно начинаться и заканчиваться символами без пробелов.</li>
                    <li>ip_v - может быть только одно из двух значений "IPv4" или "IPv6".</li>
                    <li>type - значения в списке не должны повторяться. Значение должно быть цифрой от 0 до 255.  </li>
                </ul>
