                            #### Пример использования

                            ```yaml
                            name: sg-example-1
                            rules:
                                ingress:
                                    - sgroupSet:
                                        - sg-example-2
                                    logs:  true
                                    trace: true
                                    access:
                                        tcp:
                                        - description: ""
                                            ports_from:
                                            - 64231
                                            ports_to:
                                            - 443
                                            - 80
                                        udp:
                                        - description: ""
                                            ports_from:
                                            - 64231
                                            ports_to:
                                            - 443
                                            - 80        
                            ```

                            #### Ограничения

                            <ul>
                                <li>Необходимо указать минимум одно значение в cidrSet.</li>
                                <li>Не должно быть пересечений в cidrSet.</li>
                                <li>Может быть использован только один протокол для текущего правила (TCP или UDP)</li>
                                <li class="text-justify">Значения `rules.ingress[].tcp/udp.ports_to[]` и `rules.ingress[].tcp/udp.ports_from[]` портов должно находиться в интервале от 1 до 65535. Если значение не будет указано то будет использоваться весь диапазон портов.</li>
                                <li class="text-justify">Значения `rules.ingress[].tcp/udp.ports_from[]` портов не должно пересекаться в рамках текущего правила.</li>
                            </ul>

                            ##### Пример использования

                            ```hcl
                            resource "sgroups_ie_rules" "rules" { 
                                items = { 
                                    "tcp:sg(sg-example-1)sg(sg-example-2)ingress" = {
                                        traffic     = "ingress"
                                        transport   = "tcp"
                                        sg          = "sg-example-2"
                                        sg_local    = "sg-example-1"
                                        ports = [
                                            {
                                                d = "443,80"
                                                s = "64231"
                                            }
                                        ]
                                        logs  = true
                                        trace = true
                                    }
                                    "udp:sg(sg-example-1)sg(sg-example-2)ingress" = {
                                        traffic     = "ingress"
                                        transport   = "udp"
                                        sg          = "sg-example-2"
                                        sg_local    = "sg-example-1"
                                        ports = [
                                            {
                                                d = "443,80"
                                                s = "64231"
                                            }
                                        ]
                                        logs  = true
                                        trace = true
                                    }                
                                }
                            }        
                            ```

                            #### Ограничения

                            <ul>
                                <li>Каждое правило должно иметь уникальный ключ, чтобы избежать пересечений.</li>
                                <li class="text-justify">Структура формы items должна соответствовать определенной последовательности которой необходимо придерживаться `"${transport}:cidr(${cidr})sg(${sg_name})${traffic}"`</li>
                                <li class="text-justify">sg_name - длина значения не должна превышать 256 символов. Значение должно начинаться и заканчиваться символами без пробелов.</li>
                                <li class="text-justify">Значения s (source) портов не должно пересекаться в рамках одного правила состоящего из уникального составного ключа (transport, sg_name, traffic).</li>
                                <li class="text-justify">Значения s (source) и d (distinct) портов должно находиться в интервале от 1 до 65535. Если значение не будет указано то будет использоваться весь диапазон портов.</li>
                                <li class="text-justify">Значение CIDR не должно пересекаться в рамках одного правила состоящего из уникального составного ключа (transport, sg_name, traffic).</li>
                            </ul>

                            #### Ограничения

                            <ul>
                                <li>Каждое правило должно иметь уникальный ключ, чтобы избежать пересечений.</li>
                                <li class="text-justify">Структура формы items должна соответствовать определенной последовательности которой необходимо придерживаться `"${transport}:cidr(${cidr})sg(${sg_name})${traffic}"`</li>
                                <li class="text-justify">sg_name - длина значения не должна превышать 256 символов. Значение должно начинаться и заканчиваться символами без пробелов.</li>
                                <li class="text-justify">Значения s (source) портов не должно пересекаться в рамках одного правила состоящего из уникального составного ключа (transport, sg_name, traffic).</li>
                                <li class="text-justify">Значения s (source) и d (distinct) портов должно находиться в интервале от 1 до 65535. Если значение не будет указано то будет использоваться весь диапазон портов.</li>
                                <li class="text-justify">Значение CIDR не должно пересекаться в рамках одного правила состоящего из уникального составного ключа (transport, sg_name, traffic).</li>
                            </ul>
