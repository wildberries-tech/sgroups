# Sgroup to Sgroup (Ingress/Egress)

#### Описание
<div class="paragraph">
    Правило `IE:S2C` (Ingress/Egress Security Group to CIDR) предоставляет возможность гибкого управления и контроля за сетевым трафиком между различными группами безопасности и подсетями, 
    используя TCP, UDP и ICMP протоколы. Это правило дает возможность детально настроить, какой трафик разрешен к передаче между группами безопасности и определенными подсетями, 
    тем самым гарантируя высокий уровень защиты и управления сетевой инфраструктурой.
</div>

        <h4 class="custom-heading">Ограничения</h4>
        
        <ul>
            <li class="text-justify">`name`:</li>
                <ul>
                    <li class="text-justify">Длина одного значения из списка не должна превышать 256 символов.</li>
                    <li class="text-justify">Значение должно начинаться и заканчиваться символами без пробелов.</li>
                </ul>
            <li class="text-justify">`cidrSet`:</li>
                <ul>
                    <li class="text-justify">В пределах одной Security Group и направления трафика ($traffic), необходимо обеспечить, непересекаемость диапазонов адресов подсетей.</li>
                    <li class="text-justify">Подсеть должна соответствовать формату записи, определенному в RFC 4632.</li>
                </ul>
            <li class="text-justify">`rules.$traffic[].access.$transport[].ports_to[]`:</li>
                <ul>
                    <li class="text-justify">Значения портов должно находиться в интервале от 1 до 65535.</li>
                    <li class="text-justify">Если значение не будет указано то будет использоваться весь диапазон портов.</li>
                    <li class="text-justify">Не допускаются пересечения портов в правилах в рамках одной пары Security Group.</li>
                </ul>
            <li class="text-justify">`rules.$traffic[].access.$transport[].ports_from[]`:</li>
                <ul>
                    <li class="text-justify">Значения портов должно находиться в интервале от 1 до 65535.</li>
                    <li class="text-justify">Если значение не будет указано то будет использоваться весь диапазон портов.</li>
                    <li class="text-justify">Не допускаются пересечения портов в правилах в рамках одной пары Security Group.</li>
                </ul>
            <li class="text-justify">`rules.$traffic[].access.$transport[].type[]`:</li>
                <ul>
                    <li class="text-justify">Значение должно быть числом в диапазоне от 0 до 255.</li>
                    <li class="text-justify">Повторения значений в списке не допускаются.</li>
                </ul>
        </ul>

        <h4 class="custom-heading">Ограничения</h4>

        <ul>
            <li class="text-justify">`items`:</li>
                <ul>
                    <li class="text-justify">Каждое правило должно обладать уникальным ключом для предотвращения конфликтов.</li>
                </ul>
            <li class="text-justify">`items.$ruleName`:</li>
                <ul>
                    <li class="text-justify">Форма ruleName должна быть организована в соответствии с определенной последовательностью, которую нужно соблюдать `"${$transport}:cidr(${cidr})sg(${sg_name})${traffic}"`.</li>
                </ul>
            <li class="text-justify">`items.$ruleName.sg_name`:</li>
