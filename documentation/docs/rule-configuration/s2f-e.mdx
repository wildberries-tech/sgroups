# Sgroup to FQDN (Egress)

Правило E:S2F (Egress Security Group to FQDN) предоставляет возможность гибкого управления и контроля за сетевым трафиком между различными группами безопасности и FQDN записями, используя TCP, UDP протоколы. Это правило дает возможность детально настроить, какой трафик разрешен к передаче между группами безопасности и определенными FQDN записями, тем самым гарантируя высокий уровень защиты и управления сетевой инфраструктурой.

        #### Ограничения

        <ul>
            <li class="text-justify">`name`:</li>
                <ul>
                    <li class="text-justify">Длина одного значения из списка не должна превышать 256 символов.</li>
                    <li class="text-justify">Значение должно начинаться и заканчиваться символами без пробелов.</li>
                </ul>
            <li class="text-justify">`fqdnSet`:</li>
                <ul>
                    <li class="text-justify">Длина одного значения из списка не должна превышать 256 символов.</li>
                    <li class="text-justify">Необходимо указать минимум одно значение в fqdnSet.</li>
                    <li class="text-justify">FQDN должна соответствовать формату записи, определенному в RFC `1034,1035,1123`.</li>  
                </ul>
            <li class="text-justify">`items.$ruleName.protocols[]`:</li>
                <ul>
                    <li class="text-justify">Протоколы уровня приложения, где ищут совпадения FQDN в заголовках.</li>
                    <li class="text-justify">Не допускаются дубликация протоколов в списке.</li>
                </ul>
            <li class="text-justify">`rules.$traffic[].access.$transport[].ports_to[]`:</li>
                <ul>
                    <li class="text-justify">Значения портов должно находиться в интервале от 1 до 65535.</li>
                    <li class="text-justify">Если значение не будет указано то будет использоваться весь диапазон портов.</li>
                    <li class="text-justify">Не допускаются пересечения портов в правилах в рамках одной пары Security Group.</li>
                </ul>
            <li class="text-justify">`rules.$traffic[].access.$transport[].ports_from[]`:</li>
                <ul>
                    <li class="text-justify">Значения портов должно находиться в интервале от 1 до 65535.</li>
                    <li class="text-justify">Если значение не будет указано то будет использоваться весь диапазон портов.</li>
                    <li class="text-justify">Не допускаются пересечения портов в правилах в рамках одной пары Security Group.</li>
                </ul>

        </ul>

        #### Ограничения

        <ul>
            <li class="text-justify">`items`:</li>
                <ul>
                    <li class="text-justify">Каждое правило должно обладать уникальным ключом для предотвращения конфликтов.</li>
                </ul>
            <li class="text-justify">`items.$ruleName`:</li>
                <ul>
                    <li class="text-justify">Форма ruleName должна быть организована в соответствии с определенной последовательностью, которую нужно соблюдать `"${$transport}:sg(${sg_from})fqdn$({fqdn})"`.</li>
                </ul>
            <li class="text-justify">`items.$ruleName.protocols[]`:</li>
                <ul>
                    <li class="text-justify">Протоколы уровня приложения, где ищут совпадения FQDN в заголовках.</li>
                    <li class="text-justify">Не допускаются дубликация протоколов в списке.</li>
                </ul>
            <li class="text-justify">`items.$ruleName.sg_from`:</li>
                <ul>
                    <li class="text-justify">Длина одного значения из списка не должна превышать 256 символов.</li>
                    <li class="text-justify">Значение должно начинаться и заканчиваться символами без пробелов.</li>
                </ul>
            <li class="text-justify">`items.$ruleName.fqdn`:</li>
                <ul>
                    <li class="text-justify">Длина одного значения из списка не должна превышать 256 символов.</li>
                    <li class="text-justify">Необходимо указать минимум одно значение в fqdnSet.</li>
                    <li class="text-justify">FQDN должна соответствовать формату записи, определенному в RFC `1034,1035,1123`.</li>
                </ul>
            <li class="text-justify">`items.$ruleName.ports[].s`:</li>
                <ul>
                    <li class="text-justify">Значения портов должно находиться в интервале от 1 до 65535.</li>
                    <li class="text-justify">Значения портов прописываются по одному или интервально используя `-` через запятую `80,443,8080-9090`.</li>
                    <li class="text-justify">Если значение не будет указано то будет использоваться весь диапазон портов.</li>
                    <li class="text-justify">Не допускаются пересечения портов в правилах в рамках одной пары Security Group.</li>
                </ul>
            <li class="text-justify">`items.$ruleName.ports[].d`:</li>
                <ul>
                    <li class="text-justify">Значения портов должно находиться в интервале от 1 до 65535.</li>
                    <li class="text-justify">Значения портов прописываются по одному или интервально используя `-` через запятую `80,443,8080-9090`.</li>
                    <li class="text-justify">Если значение не будет указано то будет использоваться весь диапазон портов.</li>
                    <li class="text-justify">Не допускаются пересечения портов в правилах в рамках одной пары Security Group.</li>
                </ul>
        </ul>

            <li class="text-justify">`sgFrom`:</li>
                <ul>
                    <li class="text-justify">Длина одного значения из списка не должна превышать 256 символов.</li>
                    <li class="text-justify">Значение должно начинаться и заканчиваться символами без пробелов.</li>
                </ul>
            <li class="text-justify">`protocols[]`:</li>
                <ul>
                    <li class="text-justify">Протоколы уровня приложения, где ищут совпадения FQDN в заголовках.</li>
                    <li class="text-justify">Не допускаются дубликация протоколов в списке.</li>
                </ul>
            <li class="text-justify">`rules.$traffic[].access.$transport[].ports_to[]`:</li>
                <ul>
                    <li class="text-justify">Значения портов должно находиться в интервале от 1 до 65535.</li>
                    <li class="text-justify">Если значение не будет указано то будет использоваться весь диапазон портов.</li>
                    <li class="text-justify">Не допускаются пересечения портов в правилах в рамках одной пары Security Group.</li>
                </ul>
            <li class="text-justify">`rules.$traffic[].access.$transport[].ports_from[]`:</li>
                <ul>
                    <li class="text-justify">Значения портов должно находиться в интервале от 1 до 65535.</li>
                    <li class="text-justify">Если значение не будет указано то будет использоваться весь диапазон портов.</li>
                    <li class="text-justify">Не допускаются пересечения портов в правилах в рамках одной пары Security Group.</li>
                </ul>
        </ul>
        <h4 class="custom-heading">Диаграмма последовательности</h4>

            ```mermaid
        sequenceDiagram
        participant user as User
        participant server as Server
        participant db as Database

        user->>server: Добавить FQDN правило

        alt Ошибка в запросе
            server-->>user: Показать ошибку в запросе 
        end

        server->>db: Отправить запрос
        db->>db: Проверка входящего запроса

        alt Нет такого значения имени SG
            db-->>server: Ответ с ошибкой
            server-->>user: Показать ошибку в запросе
        end

        alt Пересечение значений source портов
            db-->>server: Ответ с ошибкой
            server-->>user: Показать ошибку в запросе
        end

        db->>db: Сохранить данные о FQDN в БД

        db-->>server: Ответ с успешно добавленным новым FQDN правилом
        server-->>user: Успешно добавленное новое FQDN правило
        ```

        }
