<TabItem value='docker'>

  <dev>
    Перед установкой пользователь должен решить, где hbf-server будет хранить данные. Существует два вариата: 
    <ul>
        <li>
          In Memory.
        </li>
        <li>
          Postgres: пользователю необходимо перед запуском hbf-server выполнить [миграцию базы данных](./goose.mdx).
        </li>
    </ul>
  </dev>

  <dev>
    Перед клонированием репозитория необходимо настроить следующие переменные окружения:
    <ul>
      <li>
          <p>
            `HBF_SERVER` - IP-адрес hbf-server.
            ```bash title='Setup HBF_SERVER'
            export HBF_SERVER=0.0.0.0
            ```
          </p>
      </li>
      <li>
          <p>
            `HBF_SERVER_PORT` - порт, используемый hbf-server.
            ```bash title='Setup HBF_SERVER_PORT'
            export HBF_SERVER_PORT=9650
            ```
          </p>
      </li>
      <li>
          <p>
            `VERSION` - версия агента, которую пользователь хочет установить.
            ```bash title='Setup VERSION'
            export VERSION=1.9.1
            ```
          </p>
      </li>
      <li>
        <p>
          `SG_STORAGE_TYPE` - указать тип хранилища для hbf-agent. Допустимые значени: internal или postgres.
          
          В случае выбора типа хранилища "In Memory" установите переменную окружения командой:
          ```bash title='Setup SG_STORAGE_TYPE'
          export SG_STORAGE_TYPE="internal"
          ```
          В случае выбора типа хранилища "Postgres" установите переменную окружения командой:
          ```bash title='Setup SG_STORAGE_TYPE'
          export SG_STORAGE_TYPE="postgres"
          ```
        </p>
      </li>
      <li>
        <p>
          `SG_STORAGE_POSTGRES_URL` - URL для подключения к Postgres базе данных. Для корректного формирования 
          воспользуйтесь командой ниже:
          ```bash title='Setup SG_STORAGE_POSTGRES_URL'
          cat <<EOF > setup.sh
          #!/bin/bash
          SG_DB_USER="user"
          SG_DB_PASSWORD="password"
          SG_DB_URL="localhost:5432"
          SG_DB_NAME="swarm"

          SG_STORAGE_POSTGRES_URL="postgres://${SG_DB_USER}:${SG_DB_PASSWORD}@${SG_DB_URL}/${SG_DB_NAME}?sslmode=disable"

          export SG_STORAGE_POSTGRES_URL=$SG_STORAGE_POSTGRES_URL
          EOF
          sh setup.sh
          ```
        </p>
      </li>
    </ul>
  </dev>

  <dev>
    Клонируем репозиторий, переходим в директорию репозитория и переключаемся на тег необходимой версии.
  </dev>

  ```bash title='Git setup'
  git clone https://github.com/H-BF/sgroups
  cd sgroups
  git checkout tags/v${VERSION}
  ```

  ```bash title='Configuration file setup'
  cat <<EOF > config-server.yaml
  ---
  logger:
    # log level
    level: INFO

  metrics:
    # enable api metrics
    enable: true

  healthcheck:
    # enables|disables health check handler
    enable: true

  server:
    # server endpoint
    endpoint: tcp://${HBF_SERVER}:${HBF_SERVER_PORT}
    # graceful shutdown period
    graceful-shutdown: 30s
  EOF
  ```
  
  <dev>
    Собираем образ hbf-server воспользовавшись уже готовым Dockerfile.
  </dev>

  ```bash
  docker build -f docker/Dockerfile.server --tag sgroups:v$(cat VERSION) .
  ```

  <dev>
      В случае запуска в режиме "In Memory" выполните команду:
  </dev>

  ```bash
  docker run \
  -d \
  -v ./config-server.yaml:/opt/swarm/etc/sgroups/config-server.yaml \
  --name hbf-server \
  --entrypoint "/app/bin/sgroups" \
  --env SG_STORAGE_TYPE \
  sgroups:v$(cat VERSION) -config /opt/swarm/etc/sgroups/config-server.yaml
  ```

  <dev>
      В случае запуска в режиме "Postgres" выполните команду::
  </dev>

  ```bash
  sh setup.sh
  docker run \
  -d \
  -v ./config-server.yaml:/opt/swarm/etc/sgroups/config-server.yaml \
  --name hbf-server \
  --entrypoint "/app/bin/sgroups" \
  --env SG_STORAGE_POSTGRES_URL \
  sgroups:v$(cat VERSION) -config /opt/swarm/etc/sgroups/config-server.yaml
  ```

<TabItem value='deb'>

{' '}
  <dev>
    Перед установкой убедитесь, что вы корректно указали версию, установив переменную `VERSION` без символа 'v'.
  </dev>

  ```bash title='Environment setup'
  export HBF_SERVER=0.0.0.0
  export HBF_SERVER_PORT=9650
  export VERSION=1.9.1
  export PACKAGE_TYPE=deb
  export URL=https://github.com/H-BF/sgroups/releases/download
  export RELEASE=v$VERSION/sgroups-$VERSION-any.$PACKAGE_TYPE
  ```

  ```bash title='Install deb package'
  sudo wget -O /tmp/sgroups.deb $URL/$RELEASE
  sudo dpkg -i /tmp/sgroups.deb
  ```

  ```bash title='Server service setup'
  cat <<EOF > /etc/systemd/system/hbf-server.service
  [Unit]
  Description=sgroups
  Documentation=TODO:
  After=network.target

  [Service]
  EnvironmentFile=/opt/swarm/etc/sgroups/env
  ExecStart=/opt/swarm/sbin/sgroups --config=/opt/swarm/etc/sgroups/config.yaml
  Restart=always
  RestartSec=5
  Delegate=yes
  KillMode=process
  OOMScoreAdjust=-999
  LimitNOFILE=1048576
  LimitNPROC=infinity
  LimitCORE=infinity

  [Install]
  WantedBy=multi-user.target
  EOF
  ```

  ```bash title='Setup server service'
  cat <<EOF > /opt/swarm/etc/sgroups/config.yaml
  ---
  logger:
    # log level
    level: INFO

  metrics:
    # enable api metrics
    enable: true

  healthcheck:
    # enables|disables health check handler
    enable: true

  server:
    # server endpoint
    endpoint: tcp://${HBF_SERVER}:${HBF_SERVER_PORT}
    # graceful shutdown period
    graceful-shutdown: 30s
  EOF
  ```

  <dev>
    В случае запуска в режиме "Postgres" установите переменную `SG_STORAGE_POSTGRES_URL` командой:
  </dev>

    ```bash title="Environment setup"
    export SG_DB_USER="user"
    export SG_DB_PASSWORD="password"
    export SG_DB_URL="localhost:5432"
    export SG_DB_NAME="swarm"
    ```

    ```bash title="Environment file setup"
    cat <<EOF > /opt/swarm/etc/sgroups/env
    #!/bin/bash
    SG_STORAGE_POSTGRES_URL="postgres://${SG_DB_USER}:${SG_DB_PASSWORD}@${SG_DB_URL}/${SG_DB_NAME}?sslmode=disable"
    EOF
    ```

  <dev>
    Далее, независимо от выбраного режима хранения данных, запустите сервис hbf-server.service командой:
  </dev>
  ```bash title='Server service start'
  systemctl enable hbf-server.service
  systemctl start hbf-server.service
  ```

## Настройка

Для настройки сервера требуется использовать конфигурационный файл, который содержит поля, позволяющие настраивать параметры в соответствии с потребностями пользователей.

<div class="scrollable-x">
  <table>
    <caption>Параметры файла config-server.yaml</caption>
    <thead>
      <tr>
        <th>Параметр</th>
        <th>Описание</th>
        <th>Тип</th>
        <th>Обязательный параметр</th>
        <th>Переменная среды</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td>`logger.level`</td>
        <td>Уровень логгирования. Допустимые значения: `TRACE`, `DEBUG`, `INFO`, `WARN`, `ERROR`, `FATAL`, `PANIC`.</td>
        <td>String</td>
        <td>Нет</td>
        <td>SG_LOGGER_LEVEL</td>
      </tr>
      <tr>
        <td>`server.endpoint`</td>
        <td>IP адрес и порт hbf-сервер. Рекомендуемое значение: `0.0.0.0:9650`.</td>
        <td>String</td>
        <td>Да</td>
        <td>SG_SERVER_ENDPOINT</td>
      </tr>
      <tr>
        <td>`server.graceful-shutdow`</td>
        <td>
          Определяет период времени, в течение которого процесс сервера должен корректно завершиться. По истечению
          периода, процесс завершиться с кодом 1. Значение по умолчанию: `10s`
        </td>
        <td>String</td>
        <td>Нет</td>
        <td>SG_SERVER_GRACEFUL_SHUTDOW</td>
      </tr>
      <tr>
        <td>`telemetry.metrics.enable`</td>
        <td>
          Включить/Отключить возможность получения метрик. Значение по умолчанию: `true`. Пример получения метрик: `curl
          0.0.0.0:9650/metrics`.
        </td>
        <td>Boolean</td>
        <td>Нет</td>
        <td>SG_METRICS_ENABLE</td>
      </tr>
      <tr>
        <td>`healthcheck.enable`</td>
        <td>
          Включить/Отключить возможность получения информации о статусе hbf-сервера. Значение по умолчанию: `true`.
          Пример получения статуса: `curl 0.0.0.0:9650/healthcheck`.
        </td>
        <td>Boolean</td>
        <td>Нет</td>
        <td>SG_HEALTHCHECK_ENABLE</td>
      </tr>
      <tr>
        <td>`storage.type`</td>
        <td>
          Подключаемый тип базы данных. Значение по умолчанию: `internal`. Допустимые значения: `internal`, `postgres`.
        </td>
        <td>String</td>
        <td>Нет</td>
        <td>SG_STORAGE_TYPE</td>
      </tr>
      <tr>
        <td>`storage.postgres.url`</td>
        <td>URL для подключения к базе данных Postgres.</td>
        <td>String</td>
        <td>
          При использовании `storage.type:postgres`, данный параметр должен иметь корректный URL подключения. В ином
          случае, параметр необязательный для заполнения.
        </td>
        <td>SG_STORAGE_POSTGRES_URL</td>
      </tr>
    </tbody>
  </table>
</div>
